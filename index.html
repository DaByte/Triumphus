Functions

generate_name ()
This function generates a random name for a region. The names are taken from a seed located in the setting named “RegionNames”. This seed must consist of comma-separated names. The last name must have a comma after it.

Returns:

    string A random name for a region.
create_city (territory, important, name)
This function creates a city. It should not be used directly; instead, the CreateCity bindable function located inside territory objects should be used.

Parameters:

  • territory Part This should be the territory the city will be part of.
  • important optional bool This should be a boolean value that indicates whether the city that will be created is important. If not specified, it will be assumed that the city is not important.
  • name optional string This is a parameter that indicates the name of the city. If none is mentioned, the name “City” will be used.

Returns:

    Part A Part object that represents the city.
generate_city (territory)
This function generates a city. It should not be used directly; instead, the GenerateCity bindable function located inside territory objects should be used. This function will generate a city with a random importance, a random position (not implemented) and a random name (not implemented).

Parameters:

  • territory Part This should be the territory the city will be part of.

Returns:

    Part A Part object that represents the city.
create_territory (country, size, position, name)
This function creates a territory. It should not be used directly; instead, the CreateTerritory bindable function located inside country objects should be used.

Parameters:

  • country Model This should be the country the territory will be part of.
  • size Vector2 This should be the size of the territory.
  • position Vector2 This should be the position of the territory.
  • name optional string This is a parameter that indicates the name of the territory. If none is mentioned, the name “Territory” will be used.

Returns:

    Part A Part object that can contain parts representing cities and parts or models representing geographical elements.
create_country (name, map)
This function creates a country. It should not be used directly; instead, the CreateCountry bindable function located inside map objects should be used.

Parameters:

  • name string This should be the name of the country that will be created. This parameter is mandatory, but if it is not specified, the name “Country” will be used.
  • map Model This should be the map the country will be part of.

Returns:

    Model A Model object of which the Name property is the name of the country. The model can contain parts representing territories.
create_map (name, parent)
This function creates a map.

Parameters:

  • name string This should be the name of the map that will be created. This parameter is mandatory, but if it is not specified, the name “Map” will be used.
  • parent optional Workspace This should be either the value nil, if the map is not meant to be used yet, or the root of the physical game hierarchy.

Returns:

    Model A Model object of which the Name property is the name of the map and that can contain Model objects representing countries.
